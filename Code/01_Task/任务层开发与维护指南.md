# 任务层开发与维护指南

## 当前主干职责

当前主干 `Task` 层只保留 `task_init.h/.c`。

- `InitTask` 仍然是系统装配入口。
- 当前默认实现只保留启动闸门与一次性初始化框架。
- 旧 PCB 绑定和旧业务任务已经移出主干，不再参与默认编译。

## 主干与归档的边界

主干目录：`Code/01_Task`

- 只保留当前版本必须参与运行链的任务框架。
- 本阶段保留 `task_init`，用于承接下一版 PCB 的集中装配逻辑。

归档目录：`99_Archive/01_Task`

- 存放旧任务实现和历史案例。
- 默认不参与构建。
- 需要复用时，应按需迁回或参考重写，而不是直接恢复整个旧任务集合。

## 后续接手建议

1. 新 PCB 的硬件绑定、模块上下文注入、上电安全态，统一写入 `task_init.c` 的平台装配钩子。
2. 新业务任务按当前项目需要新建，不直接复用旧任务名，避免把历史语义带回主干。
3. 若后续 снова需要历史任务，可先从 `99_Archive/01_Task` 评估可复用片段，再决定是否正式恢复。
