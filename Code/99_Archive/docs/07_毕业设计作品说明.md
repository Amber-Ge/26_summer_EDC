# 毕业设计作品说明
## 1. 这份文档的定位
这份文档并非面向“第一次上手 STM32 的新手”，而是专门从“毕业设计作品展示”的角度说明本工程。

它重点回答下面几个问题：

1. 这个作品要展示什么。
2. 它相较于单纯功能拼接代码，工程上的价值在哪里。
3. 当前作品已经完成到什么程度。
4. 它和“通用驱动库示例工程”之间是什么关系。

如果阅读目标是复现和学习，可优先看：

1. [01_新手从零开始复现工程.md](01_新手从零开始复现工程.md)
2. [06_系统启动流程与运行逻辑总览.md](06_系统启动流程与运行逻辑总览.md)

## 2. 毕设作品的主题定位
当前代码可以作为毕业设计作品展示，主题可以概括为：

**基于 STM32F407ZGT6 的分层式嵌入式控制系统设计与实现**

在这一题目下，展示重点不应只是“让小车动起来”，而应包括：

1. 我完成了一套完整的嵌入式软件架构。
2. 这套架构可以把硬件驱动、设备模块、任务调度和控制逻辑分层管理。
3. 它不仅能运行，还能维护、扩展、复现。
4. 它能支撑多外设协同，包括底盘控制、循迹、步进、显示、调试和视觉通信。

## 3. 本作品的硬件与功能基础
### 3.1 主控与平台

1. 主控芯片：`STM32F407ZGT6`
2. 硬件形式：`反客科技开发板 + 自制扩展 PCB`
3. 开发方式：`STM32CubeMX + CMake + CLion + FreeRTOS`

### 3.2 接入的主要设备

1. 12 路灰度循迹传感器
2. 轮趣 `MG513X` 直流电机
3. 张大头闭环步进电机一体驱动设备
4. `K230`
5. OLED
6. 蜂鸣器与激光继电器
7. 编码器、电池电压采样、按键和 VOFA 调试接口

## 4. 毕设角度最应该展示的“成果”
### 4.1 展示重点应是系统化能力

答辩展示时，不应把重点放在“某一个函数怎么写”，而应重点说明下面这些系统能力：

1. 建立了可维护的分层软件架构
2. 实现了基于 `FreeRTOS` 的多任务协同
3. 实现了直流电机闭环控制链路
4. 实现了 12 路灰度循迹输入到控制输出的完整链路
5. 实现了 `K230 -> 双轴步进` 的通信与控制链路
6. 建立了 OLED、VOFA、按键反馈、电压采样等完整辅助系统

### 4.2 建议重点强调的工程亮点

当前作品最应强调的，并不是“代码量多”，而是以下几点：

1. 使用 `Task -> Module -> Driver -> HAL` 的明确分层。
2. 使用 `InitTask` 完成统一装配，而不是到处散写初始化。
3. 使用模块上下文和绑定表，让引脚映射与业务逻辑解耦。
4. 使用 `mod_uart_guard + tx_mutex` 处理串口资源仲裁。
5. 使用 `FreeRTOS` 信号量和互斥锁实现任务之间的解耦通信。

## 5. 作品当前已经具备的能力
从当前代码实现来看，可以明确、稳妥地说已经具备：

1. 系统启动、任务创建、资源同步机制完整。
2. 底盘直流电机控制链路完整。
3. 直线模式控制链路完整。
4. 基础循迹模式控制链路完整。
5. 双轴步进串口控制链路完整。
6. `K230` 自定义协议接收与误差解析链路完整。
7. OLED、VOFA、按键反馈、继电器控制等辅助功能完整。
8. 工程可以用 `CMake + CLion` 方式构建，而不是只依赖某个 IDE 工程文件。

## 6. 作品当前的边界必须讲清楚
这一部分非常重要。  
在毕设说明里必须主动讲清边界，否则评审很容易默认项目已经覆盖了更大的能力范围。

### 6.1 建议采用的表述方式

本工程可以作为：

1. 毕业设计作品展示
2. 通用驱动库与分层软件架构展示
3. 面向循迹和直线控制的基础验证平台
4. 面向视觉误差驱动双轴步进联调的平台

但当前不应夸大为：

1. 完整覆盖某一具体电赛赛题的最终成品
2. 所有比赛动作均已完成并稳定验证
3. 所有复杂运动控制和高层策略都已经闭环完善

### 6.2 更适合现在代码状态的能力边界

最合理的表达是：

1. 重点保证基础循迹
2. 重点保证走直线
3. 重点保证双轴步进控制链路能够建立并联调
4. 重点展示分层驱动库、模块化设计与维护能力

## 7. 为什么这个作品有“毕业设计价值”
单纯让电机转起来，不足以构成一个有质量的毕设。  
本作品的主要工程价值在于：

1. 它不是一次性脚本式代码，而是一套可持续维护的工程。
2. 它把硬件接线、模块绑定、任务逻辑、控制算法和通信协议分离开了。
3. 后续维护者可以在现有基础上继续扩展功能，而不是推倒重来。
4. 这已经体现出“工程化设计能力”，而不是只会拼接 HAL 函数。

## 8. 可以在答辩中重点讲的技术路线
### 8.1 开发工具链路线

可重点说明：

1. 使用 `STM32CubeMX` 完成底层外设配置
2. 使用 `CMake` 管理工程构建
3. 使用 `CLion` 作为主要开发环境
4. 使用 `ST-LINK` 进行下载与调试
5. 使用 `VOFA` 做联调可视化

### 8.2 软件架构路线

可重点说明：

1. `CubeMX` 负责生成底层初始化
2. `Driver` 负责最小硬件抽象
3. `Module` 负责设备语义和协议封装
4. `Task` 负责状态机、时序与多模块协作
5. `FreeRTOS` 负责调度和同步

### 8.3 控制路线

可重点说明：

1. 双直流电机底盘采用 PWM + 编码器反馈
2. 直线控制采用外环位置、内环速度的结构
3. 循迹控制通过 12 路灰度传感器计算权重偏差
4. 步进控制通过视觉误差驱动双轴位置调整

## 12. 一段适合写在毕设说明中的总结
可采用下面这个思路来总结本作品：

本项目基于 `STM32F407ZGT6` 搭建了一套面向运动控制与多外设协同的嵌入式软件系统。  
系统采用 `Task -> Module -> Driver -> HAL` 的分层架构，在 `FreeRTOS` 调度框架下完成了底盘直流电机控制、12 路灰度循迹、K230 自定义协议通信、双轴步进控制、OLED 状态显示、VOFA 调试以及按键反馈等功能模块的集成。  
该工程不仅能够作为毕业设计作品展示系统设计与实现能力，也具备较好的复现性、维护性和二次开发价值。
