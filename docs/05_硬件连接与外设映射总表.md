# 硬件连接与外设映射总表
## 1. 这份文档解决什么问题
这份文档专门解决 3 类问题：

1. 读者拿到这套工程之后，不知道每个模块到底接在 `STM32F407ZGT6` 的哪个引脚上。
2. 即使能够看懂代码，也不清楚“代码中的模块绑定”和“项目 PCB 上的接线”是如何一一对应的。
3. 后续如果需要改引脚、换板子或换外设，不知道应该先改硬件、先改 `.ioc`，还是先改 `task_init.c`。

这份文档的定位是“硬件映射总表”。  
如果需要的是：

1. 从零安装软件并把工程跑起来，请看 [01_新手从零开始复现工程.md](01_新手从零开始复现工程.md)
2. 在 `CLion` 里正确构建和调试，请看 [02_CLion环境配置与CMake构建指南.md](02_CLion环境配置与CMake构建指南.md)
3. 改引脚并重新生成代码，请看 [04_CubeMX引脚修改与工程维护指南.md](04_CubeMX引脚修改与工程维护指南.md)
4. 看系统从上电到业务运行的完整逻辑，请看 [06_系统启动流程与运行逻辑总览.md](06_系统启动流程与运行逻辑总览.md)

## 2. 文档依据
本文件内容来自以下几部分交叉核对：

1. 工程源码中的 `.ioc` 外设配置。
2. `Core/Src/*.c` 中 `GPIO / USART / TIM / ADC / I2C` 初始化代码。
3. `Code/01_Task/Src/task_init.c` 中的模块绑定表。
4. 项目补充的硬件资料与引脚表。

因此，本文件优先保证“和当前代码一致”。  
如果未来项目 PCB、接线表和当前代码不一致，应以“代码实际生效配置 + 最新确认的硬件接线”重新同步更新本文件。

## 3. 当前硬件平台说明
### 3.1 主控

1. 主控芯片：`STM32F407ZGT6`
2. 使用形式：`反客科技开发板 + 项目配套扩展 PCB`
3. 工程管理方式：`STM32CubeMX + CMake + CLion + FreeRTOS`

### 3.2 当前外设与执行器

1. 12 路灰度循迹传感器
2. 轮趣 `MG513X` 直流减速电机
3. 张大头闭环步进电机一体驱动设备
4. `K230`
5. OLED 显示屏
6. 蜂鸣器继电器
7. 激光继电器
8. 3 个按键
9. 电池电压采样

### 3.3 需要先建立的认识
这套工程不是“把所有逻辑直接写在引脚上”的写法，而是 3 层映射同时成立：

1. `CubeMX/.ioc` 决定 MCU 外设和引脚复用关系。
2. `task_init.c` 决定哪个模块绑定哪个 `GPIO/TIM/UART/ADC`。
3. 业务任务决定什么时候使用这些模块。

因此看到一个引脚时，不能只在原理图里记住它，还必须知道它在代码里属于哪一层的哪个对象。

### 4.1 当前主要外设实例

| `ADC1` | 电池电压采样 | `PA2` |

| `TIM2` | 左轮编码器 | 32 位编码器模式 |
| `TIM3` | 右轮编码器 | 16 位编码器模式 |

| `UART4` | K230 | 自定义协议 |
| `UART5` | 步进 X 轴 | 串口通信 |
| `USART2` | 步进 Y 轴 | 串口通信 |
| `USART3` | VOFA 调试串口 | 115200 |

### 4.2 特殊说明

1. `TIM6` 被用作 `CubeMX` 的时间基准，不参与用户业务控制。
2. LED 是低电平有效，不是高电平有效。
3. 左右电机方向和右轮编码器方向都做过软件修正，不能只凭“看接线颜色”来判断方向正负。
4. 步进驱动和单片机串口通信时必须共地。

## 6. 12 路灰度传感器权重映射
代码里不是“只看哪个引脚亮了”，而是给每一路分配了权重，最后算一个归一化偏差值。

当前在 `task_init.c` 中配置的权重从左到右为：

| 通道顺序 | 引脚 | 权重 |
| --- | --- | --- |
| 1 | `PG0` | `0.60` |
| 2 | `PG1` | `0.40` |
| 3 | `PG5` | `0.30` |
| 4 | `PG6` | `0.20` |
| 5 | `PG7` | `0.10` |
| 6 | `PG8` | `0.05` |
| 7 | `PG9` | `-0.05` |
| 8 | `PG10` | `-0.10` |
| 9 | `PG11` | `-0.20` |
| 10 | `PG12` | `-0.30` |
| 11 | `PG13` | `-0.40` |
| 12 | `PG14` | `-0.60` |

需要理解这张表的意义：

1. 这 12 路不是“谁亮谁就直接左转右转”。
2. 它们先被转换为 `0/1` 的黑线检测状态。
3. 再按权重求偏差。
4. 最终 `task_dcc.c` 在循迹模式里根据这个偏差去修正左右轮目标速度。

如果未来调整灰度模块安装顺序，最常见的工作通常不是改算法，而是重新排列这一张权重表。

## 7. 电机控制链路说明
### 7.1 直流电机

直流电机链路由 3 部分组成：

1. 方向 GPIO
2. PWM 输出
3. 编码器反馈

在代码中的绑定关系是：

1. 左轮：
   - 方向：`PE8 + PE7`
   - PWM：`TIM4_CH1`
   - 编码器：`TIM2`
2. 右轮：
   - 方向：`PE9 + PE10`
   - PWM：`TIM4_CH2`
   - 编码器：`TIM3`

必须特别注意两件事：

1. 左轮方向线在代码绑定里做了顺序修正。
2. 右轮编码器方向在代码里做了反向修正。

这意味着：

1. 如果轮子正反方向不符合预期，不要第一反应去改 PID。
2. 先确认是“电机方向绑定错了”，还是“编码器方向反了”，还是“机械安装方向和项目定义的正方向不一致”。

### 7.2 步进电机

当前工程把双轴步进分成两个独立串口通道：

1. X 轴：`UART5`
2. Y 轴：`USART2`

任务层通过 `StepperTask` 调用 `mod_stepper` 下发协议命令。  
模块层只负责组帧和发送，不做轨迹规划。

当前软件假设：

1. 两个驱动地址都为 `1`
2. 每圈脉冲数按 `3200` 计算
3. X 轴软限位 `800 pulse`
4. Y 轴软限位 `400 pulse`

如果后续更换了驱动器参数或细分数，必须同步更新：

1. 步进任务参数
2. 相关文档
3. 联调结果记录

## 8. K230、VOFA 与 OLED 的角色
### 8.1 K230

`K230` 当前通过 `UART4` 接入。  
它不是直接控制电机，而是给 `StepperTask` 提供视觉误差帧。

当前协议模块按固定 12 字节帧解析：

1. 帧头：`AA AA`
2. 长度字段：`06`
3. 两组逻辑电机 ID 和误差值
4. 校验
5. 帧尾：`55 55`

### 8.2 VOFA

`VOFA` 通过 `USART3` 连接。  
作用主要是调试：

1. 看步进误差曲线
2. 看 DCC 状态
3. 辅助联调控制参数

### 8.3 OLED

OLED 使用 `I2C2`。  
当前主要显示：

1. 电压
2. 模式值
3. DCC 运行状态

## 9. 从接线到代码的对应关系怎么找
当需要排查某个功能，例如“黄灯为什么会亮”时，可按这个顺序往下查找：

1. 先在本文件查到黄灯对应 `PF12`
2. 再去看 `.ioc` 或 `gpio.c`，确认 `PF12` 被配置成输出
3. 再去看 `task_init.c`，确认 `PF12` 被绑定给 `mod_led` 的哪个逻辑通道
4. 再去看 `task_gpio.c`，确认哪个任务在什么时候调用了 `mod_led_on/off`

这就是后续维护这类分层工程时的标准排查方式。

## 10. 新手推荐接线顺序
不要一开始就把所有东西全部接满。  
建议按风险从低到高分阶段接线。

### 第一阶段：只接最基本的下载与显示链路

1. `ST-LINK`
2. OLED
3. 三个按键
4. 三个 LED
5. 蜂鸣器继电器

### 第二阶段：接调试与采样链路

1. VOFA 串口
2. 电池电压采样
3. 灰度传感器

### 第三阶段：接执行机构

1. 左右直流电机与编码器
2. 步进 X 轴
3. 步进 Y 轴

### 第四阶段：接视觉链路

1. K230
2. 步进联调

## 11. 如果需要改引脚，应先改哪一层
### 场景 1：只是把某个 LED 从 `PF12` 换到别的脚

通常至少要改 2 个地方：

1. `.ioc`
2. `task_init.c` 里的 LED 绑定表

### 场景 2：把 OLED 从 `I2C2` 换到别的 I2C

通常至少要改 3 个地方：

1. `.ioc`
2. `Core/Src/i2c.c`
3. `task_oled.c` 中的绑定对象

### 场景 3：把步进电机通信口换了

通常至少要改 3 个地方：

1. `.ioc`
2. `usart.c`
3. `StepperTask` 准备通道时使用的串口绑定

完整方法请直接看 [04_CubeMX引脚修改与工程维护指南.md](04_CubeMX引脚修改与工程维护指南.md)。

## 12. 首次上电前检查清单
上电前至少确认下面这些事情：

1. `ST-LINK` 接线正确。
2. 开发板和扩展板共地。
3. 步进驱动和单片机共地。
4. 电机供电和主控供电没有接反。
5. 直流电机首次联调时最好离地或空载。
6. 步进首次联调时先降低速度，确认方向。
7. 灰度传感器供电和输出电平范围与主控兼容。
8. `K230` 串口电平与波特率匹配。

## 13. 这份总表怎么维护
只要下面任意一项变化，就必须同步更新本文件：

1. `.ioc` 中外设实例变了。
2. GPIO 绑定变了。
3. `task_init.c` 的绑定表变了。
4. 传感器顺序和权重变了。
5. 电机方向修正规则变了。
6. 串口用途变了。

如需继续维护，请接着看 [09_文档维护与更新规范.md](09_文档维护与更新规范.md)。
